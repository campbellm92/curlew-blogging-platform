How to deploy your blog (with Heroku)

This brief guide will walk you through you how to host your blog/website with Heroku and have it direct to your domain. Bear in mind that Heroku is a paid service. There are alternatives, but Heroku is what I use for the moment, mostly because the app that we're deploying is essentially a Node.js app, which works well with Heroku.

You're also probably going to want to register a domain. I'm in Italy and I use IONOS, but feel free to choose whichever provider suits your budget or whatever other considerations you want to take into account.

This guide assumes there is probably a decent amount of overlap in deploying a website with providers other than Heroku and IONOS. There is also some light usage of Git and the command line, but I'll cover everything you need to know.

Without further ado, here's the guide in two parts.

Creating a Heroku app:

  1. Visit the Heroku signup page and make an account.
  2. Install Heroku:
    You'll need to install the Heroku CLI (Command Line Interface) in order to deploy your project. Instructions for various operating systems can be found here.
  3. Login to Heroku in the terminal:
    Run heroku login in the terminal and enter your login details when the window pops up.
  4. Initialise a Git repository:
    A Git repository is required to use Heroku. Run these commands in your project root directory:
    git init
    git add .
    git commit -m "initial commit"
  5. Create a Heroku app:
    In the Heroku CLI, run heroku create your-app-name.
  6. Deploy your app to Heroku:
    Then, in the terminal run git push heroku main. If that doesn't work, change 'main' to 'master'.

Connecting your Heroku app to your domain name:

  1. Add your domain:
    Find your new app in your Heroku account on the Heroku website. Go to Settings, then scroll down to the Domains section and click 'Add domain'. You don't need to put the protocol (https) or www. At this point you'll also want to copy the value that you see under 'DNS Target'. target
  2. Configure DNS:
    Your domain name provider should have a section (possibly under Settings) where you can view DNS (Domain Name System) configurations. Create a new (or edit the existing) CNAME record and paste the DNS Target value you copied from the previous step into the 'Points to' field. You shouldn't need to do anything with the other fields. Save the record.
  3. Add an SSL Certificate:
    Head back to Heroku and on the same page where you added your domain, find and click on 'Configure SSL'. Here you can choose the automatic option or add your own SSL as you see fit.

Your site should now be live! Sometimes DNS changes and SSL certificates can take a bit of time to implement, so if it doesn't work straight away, give it a bit of time and try again.

Don't forget, after making changes to your blog/site you'll need to run
git add
git commit -m "message that describes changes to your site"
git push heroku main
